## 1. 프로젝트 핵심 요약 (30초 버전)

### 한 줄 요약

**드론 항공 영상에서 노이즈, 블러, 저해상도 등 실환경 열화 조건이 객체 탐지 성능에 미치는 영향을 정량 분석하고, 3가지 강건성 개선 전략(Corruption Augmentation, Image Restoration)을 이미지와 비디오 도메인 모두에서 비교 검증한 프로젝트.**

### 자기소개서용 두 문장 요약

> "드론/항공 감시 시스템에서 실환경 열화(블러, 노이즈, 저해상도)가 객체 탐지 성능에 미치는 영향을 정량 분석하고, 3가지 강건성 전략을 이미지·비디오 두 도메인에서 체계적으로 비교 검증했습니다. 총 11개 모델 학습, 52회 평가를 통해 'Clean 성능 ≠ 실환경 강건성'임을 실험적으로 입증하였고, 운용 환경별 최적 전략 선택 가이드라인을 도출했습니다."

---

## 2. 프로젝트를 시작한 이유

방산/항공 분야에서 드론이나 감시 카메라의 객체 탐지 시스템은 이상적인 환경이 아닌 **실전 환경**에서 동작해야 한다. 카메라 진동으로 인한 블러, 센서 노이즈, 저해상도 등이 불가피한데, 대부분의 객체 탐지 연구는 깨끗한(Clean) 이미지 기준 성능만 보고한다.

**"Clean에서 잘 되는 모델이 실제로도 잘 될까?"**

이 질문에서 출발하여, 실험적으로 검증하고 개선 방법을 비교하는 것이 목표였다. 이후 이미지에서 발견한 결과가 비디오 도메인에서도 동일하게 적용되는지까지 확장 실험했다.

---

## 3. 내가 한 것 (전체 흐름)

### Phase 1: 환경 구축 및 Baseline 평가 (이미지 탐지)

- VisDrone-DET 데이터셋을 COCO/YOLO 포맷으로 변환 (6개 클래스 선별)
- 3개 모델(Faster R-CNN, RT-DETR-L, YOLOv8m) Baseline 학습
- 노이즈/블러/저해상도 열화 테스트셋 생성 (각 548장)
- 3 모델 × 4 조건 = **12회 평가** → 성능 하락 정량화

### Phase 2: Corruption Augmentation (이미지 탐지)

- 학습 시 50% 확률로 corruption을 랜덤 적용하여 강건성 향상 전략 구현
- Ultralytics 프레임워크 내부 파이프라인에 monkey-patching으로 삽입
- 3개 모델 augmented 버전 재학습 → **12회 평가** → 개선 효과 검증

### Phase 3: Image Restoration + 3-Strategy 비교 (이미지 탐지)

- 경량 U-Net (3.70M 파라미터, Residual Learning) 설계 및 학습
- PSNR 34.03dB, SSIM 0.947 달성
- 열화 이미지 복원 후 Baseline 모델 추론 파이프라인 구축
- 3 모델 × 4 조건 = **12회 평가** → 3-Strategy 종합 비교

### Phase 4: 시각화 및 문서화 (이미지 탐지)

- 36회 평가 결과를 9개 차트로 시각화 (막대, 히트맵, 레이더)
- 데모 비교 이미지 15장 생성 (Blur 조건에서 탐지 수 비교)
- 종합 보고서 작성

### Phase 5: 비디오 탐지 확장 실험 (VID)

- VisDrone-VID 비디오 시퀀스 프레임 추출 및 YOLO 포맷 변환
- YOLOv8m, RT-DETR-L 두 모델에 대해 Baseline + Augmented 학습
- DET와 동일한 테스트셋으로 **16회 평가** → 결과 비교 및 시각화 (5개 차트)
- "이미지에서의 발견이 비디오 도메인에서도 일반화됨" 검증

## 4. 핵심 수치 (면접에서 말할 숫자)

### 이미지 탐지 (VisDrone-DET)

| 질문 포인트 | 핵심 수치 |
|---|---|
| Clean 모델의 Blur 취약성 | Faster R-CNN: **0.532 → 0.287, -46.1% 폭락** |
| Augmentation 효과 (최대) | YOLOv8m Blur 하락률: **-35.1% → -7.9% (4.4배 강건)** |
| Augmentation이 Clean을 희생? | RT-DETR: 오히려 **Clean +4.2%p 향상** |
| Restoration의 Blur 강점 | YOLOv8m: 0.432 → **0.640 (+0.208)**, Augmented보다 +0.032 높음 |
| Restoration의 Noise 역효과 | YOLOv8m: **0.577 → 0.201, -65% 폭락** |
| 종합 최강 모델 | YOLOv8m_aug: 평균 하락률 **-4.7%** (3개 조건 평균) |

| 질문 포인트 | 핵심 수치 |
|---|---|
| VID Baseline의 Blur 취약성 | YOLOv8m-VID: **-38.3% 하락** |

| DET vs VID 패턴 일치 | 두 도메인 모두 동일: **Blur 최취약, Augmentation 일관 효과적** |

## 5. 이 프로젝트의 차별점

### 5-1. 단순 구현이 아닌 "실험 설계"

단순히 모델을 학습하고 성능을 측정하는 것이 아니라, **변인 통제된 실험**을 설계하였다.

- 모든 전략 비교 시 동일 하이퍼파라미터, 동일 데이터, 동일 평가셋 사용
- 차이의 원인을 오직 "전략"으로 귀속시킬 수 있음
- 이는 단순 튜토리얼 재현과 근본적으로 다른 **연구적 접근**

### 5-2. 3가지 아키텍처 패러다임을 동시 비교

Faster R-CNN(2-Stage CNN), RT-DETR(Transformer), YOLOv8(1-Stage CNN)을 동일 조건에서 비교하여 **아키텍처가 강건성에 미치는 구조적 영향**을 규명했다. 단일 모델만 사용하는 대부분의 프로젝트와 차별된다.

### 5-3. 이미지 → 비디오 도메인 일반화 검증

이미지 탐지(DET)에서 발견한 패턴이 비디오 탐지(VID)에서도 재현되는지 확인하였다. **결과의 일반화 가능성(generalizability)**을 실험적으로 검증한 것은 단순 프로젝트 구현을 넘어선 연구적 기여다.

### 5-4. 실패도 결과로 분석

Restoration이 Noise에서 역효과를 낸 결과를 "실패"로 숨기지 않고, **over-smoothing 메커니즘을 분석**하여 각 전략의 trade-off를 도출했다. 긍정적 결과만 보고하는 것이 아니라, 한계까지 명확히 규명한 것이 신뢰성을 높인다.

### 5-5. 총 52회의 체계적 정량 평가

11개 모델 × 4개 조건 = 52회 평가. 단발성 실험이 아니라 **반복 가능하고 재현 가능한(reproducible)** 실험 체계를 구축했다.

**어필 포인트**:
- 이미지 열화 특성에 대한 깊은 이해

## 7. 기술적으로 어려웠던 점과 해결 과정

### 7-1. RT-DETR GPU 디바이스 에러

**문제**: RT-DETR 모델을 GPU에서 추론할 때 `RuntimeError: Expected all tensors to be on the same device` 에러 발생.

**원인 분석**: Ultralytics의 `RTDETRDecoder` 클래스에서 `valid_mask`와 `shapes`가 **클래스 레벨 속성**으로 정의되어 있어, `.cuda()` 호출에도 CPU에 남아 있었다. 일반적인 `nn.Parameter`/`nn.Buffer`와 달리 `.to(device)`에 자동으로 이동하지 않는 파이썬 속성이었다.

**해결**: 모델 로딩 후 `m.shapes = []`로 초기화하여 CPU 텐서 참조 제거. 추론 시 GPU에서 자동 재생성되도록 함.

**배운 점**: 에러 메시지만으로는 원인을 알 수 없었고, Ultralytics 소스코드를 직접 분석해야 했다. 오픈소스 프레임워크를 "블랙박스"가 아닌 "분석 가능한 코드"로 다루는 역량을 키웠다.

---

### 7-2. Ultralytics 학습 파이프라인에 Custom Augmentation 주입

**문제**: RT-DETR과 YOLOv8은 Ultralytics 자체 학습 루프를 사용하는데, 커스텀 corruption을 삽입할 공식 인터페이스가 없었다.

**해결**: `Albumentations.__call__` 메서드를 **monkey-patching**하여, 기존 augmentation 파이프라인(mosaic, mixup 등)은 유지하면서 corruption을 추가 적용. `labels["img"]` 배열에 직접 접근.

**배운 점**: 프레임워크를 포크하지 않고도 핵심 동작을 수정하는 기법. 단, 버전 의존성이 생기므로 버전을 고정하고 문서화해야 한다.

---

### 7-3. U-Net Skip Connection 채널 불일치

**문제**: Decoder의 UpBlock에서 `RuntimeError: expected input to have 256 channels, but got 384 channels` 에러 발생.

**원인**: Decoder가 업샘플링한 텐서와 skip connection 텐서를 concat할 때 채널 수 계산 오류. `ConvBlock(out_ch * 2, out_ch)`로 가정했으나, 업샘플 채널과 skip 채널이 다를 수 있었다.

**해결**: `UpBlock` 생성자를 `(in_ch, skip_ch, out_ch)`로 분리하여 명시적 지정. `ConvBlock(in_ch + skip_ch, out_ch)`로 정확한 채널 연산.

**배운 점**: Encoder-Decoder 구조에서 각 레이어의 입출력 채널을 명확히 추적하고 문서화하는 것이 버그 예방에 필수적이다.

---

### 7-4. Restoration이 Noise에서 역효과를 낸 문제

**문제**: U-Net 복원 후 Noise 조건 탐지 성능이 Baseline보다 크게 하락 (YOLOv8m: 0.577 → 0.201).

**원인 분석**: U-Net이 노이즈 제거 과정에서 텍스처와 엣지 정보까지 과도하게 제거(over-smoothing). 소형 객체(pedestrian, motor)의 미세한 특징이 사라져 탐지 불가.

**해결/인사이트**: 이것은 "실패"가 아닌 **중요한 연구 발견**이었다. 단일 복원 모델이 서로 다른 성격의 열화(공간적 blur vs 주파수적 noise)를 동시에 최적으로 처리하기 어렵다는 본질적 한계를 실험적으로 입증. → **"전략마다 고유한 trade-off가 있으며, 운용 환경에 맞는 선택이 중요하다"**는 핵심 인사이트 도출.

---

### 7-5. Faster R-CNN의 Blur 극단적 취약성 분석

**문제**: Faster R-CNN이 Blur에서 -46.1% 하락 (다른 모델 대비 압도적으로 취약).

**원인 분석**: 2-Stage 구조의 **Cascade Failure**. 1단계 RPN이 블러된 이미지에서 영역 제안에 실패하면 2단계로 넘어갈 기회 자체가 사라진다. 에러가 단계별로 누적되는 구조적 취약점.

**배운 점**: 아키텍처 설계 자체가 강건성에 구조적 영향을 미친다. 정확도만 보는 것이 아니라, **운용 환경의 열화 특성에 맞는 아키텍처를 선택**해야 한다.

## 8. 프로젝트의 의의와 한계

### 8-1. 의의

| 의의 | 설명 |
|---|---|
| **실환경 강건성 정량화** | "Clean 성능 ≠ 실환경 강건성"을 52회 평가로 실험적 입증 |
| **전략별 trade-off 규명** | Augmentation(범용성) vs Restoration(Blur 특화/Noise 취약) |
| **아키텍처별 강건성 분석** | CNN의 cascade failure vs Transformer의 전역 어텐션 강건성 |
| **도메인 일반화 검증** | 이미지→비디오 도메인에서도 동일 패턴 재현 확인 |
| **실용적 가이드라인 도출** | 운용 환경에 따른 최적 전략 선택 기준 제시 |

### 8-2. 한계

| 한계 | 설명 |
|---|---|
| **열화 파라미터 고정** | sigma=15, kernel=9 등 단일 severity만 분석. 다양한 강도(mild/severe) 실험 필요 |
| **열화 유형 제한** | 안개, 비, 야간, 압축 아티팩트 등 방산 특화 조건 미포함 |
| **전략 조합 미실험** | Augmented + U-Net 조합이 Noise와 Blur 모두에서 최적일 가능성 있음 |
| **비디오 특화 전략 부재** | 시간적 연속성을 활용한 video-aware augmentation 미구현 |
| **실시간 성능 미측정** | 추론 지연(latency), FPS 등 실전 배포 관점의 속도 측정 없음 |

### 8-3. 향후 발전 방향

- **Adaptive Pipeline**: 열화 유형 분류 → 조건별 최적 전략 자동 선택
- **Video-aware Augmentation**: 프레임 간 temporal consistency를 활용한 VID 특화 전략
- **Noise-robust Restoration**: Perceptual loss, GAN 기반 복원으로 over-smoothing 방지
- **경량화 & 배포**: Edge 디바이스용 모델 경량화 및 TensorRT/ONNX 변환

## 9. 면접 예상 질문 및 답변

### 기본 프로젝트 설명

**Q. 이 프로젝트를 한 줄로 설명하세요.**

> 드론 항공 영상에서 실환경 열화(블러, 노이즈, 저해상도)가 객체 탐지 성능에 미치는 영향을 정량화하고, 3가지 강건성 개선 전략을 이미지·비디오 두 도메인에서 체계적으로 비교한 프로젝트입니다.

---

**Q. 왜 이 프로젝트를 했나요?**

> 방산/항공 분야의 실전 시스템은 깨끗한 이미지가 아닌 열화된 환경에서 동작해야 합니다. 그런데 대부분의 AI 벤치마크는 Clean 이미지 기준입니다. "Clean 성능이 좋으면 실환경에서도 좋을까?"라는 가설을 직접 실험으로 검증하고 싶었습니다. 결과적으로 이 가설은 **틀렸습니다.** Faster R-CNN은 Clean에서 0.532이지만 Blur 조건에서 0.287로 -46.1% 폭락했습니다.

---

**Q. 이 프로젝트의 핵심 기여가 무엇인가요?**

> 3가지 강건성 전략(Baseline, Corruption Augmentation, Image Restoration)을 동일 조건에서 52회 체계적으로 평가하여, 각 전략의 강점과 한계를 정량적으로 규명한 것입니다. 특히 ① 이미지에서 발견한 패턴이 비디오 도메인에서도 재현됨을 확인하여 결과의 일반화 가능성을 검증하였고, ② "만능 전략은 없으며 운용 환경에 맞는 선택이 중요하다"는 실용적 가이드라인을 도출했습니다.

---

### 기술적 심층 질문

**Q. 왜 이 3개 모델을 선택했나요?**

> 객체 탐지의 3가지 주요 아키텍처 패러다임을 대표하기 위해서입니다. Faster R-CNN(2-Stage CNN), YOLOv8(1-Stage CNN), RT-DETR(Transformer). 아키텍처 자체가 열화 강건성에 어떤 구조적 영향을 미치는지 비교하기 위한 선택이었고, 실제로 Transformer(RT-DETR)의 전역 어텐션이 CNN 대비 Blur에서 더 강건하다는 것을 확인했습니다.

---

**Q. Corruption Augmentation이 왜 효과적인가요?**

> 두 가지 이유입니다. 첫째, 학습 과정에서 열화된 이미지를 경험하면 모델이 열화에 불변하는(invariant) 특징을 학습합니다. 블러된 자동차의 전체적 형태와 맥락을 활용하는 법을 학습하여, 선명한 텍스처에 과도하게 의존하지 않게 됩니다. 둘째, 일종의 정규화(regularization) 효과가 있어 RT-DETR의 경우 Clean 성능까지 +4.2%p 향상되었습니다.

---

**Q. Image Restoration이 Noise에서 실패한 이유는?**

> U-Net이 노이즈를 제거하는 과정에서 객체의 텍스처와 미세한 엣지까지 함께 제거하는 over-smoothing 현상이 발생했습니다. Blur는 공간적 정보 퍼짐이라 U-Net이 역방향 복원이 가능하지만, Noise는 주파수적 랜덤 교란이라 제거 과정에서 객체 경계 정보까지 사라집니다. 특히 소형 객체(보행자, 오토바이)에서 치명적이었습니다. 단일 복원 모델이 서로 다른 성격의 열화를 동시에 최적으로 처리하기 어렵다는 본질적 한계입니다.

---

**Q. 비디오 실험을 추가한 이유는? 이미지 결과와 어떻게 달랐나요?**

> 이미지 탐지에서 발견한 패턴(Blur 최취약, Augmentation 효과적)이 비디오 도메인에서도 재현되는지 검증하고 싶었습니다. 결과적으로 **동일한 패턴이 재현**되었습니다. YOLOv8m의 Blur 하락률이 이미지에서 -35.1%→-7.9%였다면, 비디오에서는 -38.3%→-9.1%로 거의 동일한 수준의 개선을 보였습니다. 절대 성능 자체는 비디오 도메인의 더 어려운 데이터(빠른 움직임, 가려짐 등)와 도메인 갭으로 인해 이미지보다 낮았지만, 전략의 효과는 도메인을 초월해 일반화됩니다.

---

**Q. Transformer(RT-DETR)가 Blur에 강건한 이유를 설명하세요.**

> CNN은 3×3 커널로 주변 픽셀의 지역적 패턴(텍스처, 엣지)을 학습합니다. Blur가 이 지역적 패턴을 파괴하면 CNN은 특징을 잡지 못합니다. 반면 Transformer는 Self-Attention을 통해 이미지 전체의 모든 위치 간 관계를 학습합니다. "도로 위에 직립한 형태" "차선 옆에 직사각형"과 같은 전역적 맥락으로 추론할 수 있어, 일부 지역 정보가 블러로 파괴되어도 다른 위치의 정보로 보완이 가능합니다.

---

**Q. U-Net의 Residual Learning 방식을 선택한 이유는?**

> 이미지 전체를 처음부터 재생성하는 것보다, 열화된 부분만 수정하는 잔차(residual)를 학습하는 것이 훨씬 효율적이기 때문입니다. 클린 이미지와 열화 이미지의 차이(잔차)는 대부분 0에 가까우므로 학습이 안정적이고 수렴이 빠릅니다. ResNet이 깊은 네트워크 학습을 안정화한 것과 같은 원리입니다. 실제로 PSNR 34.03dB, SSIM 0.947을 달성하여 충분한 복원 품질을 확인했습니다.

---

**Q. 실제 방산 시스템에 이 결과를 적용한다면?**

> 드론의 주된 열화는 카메라 진동에 의한 Blur입니다. 이 경우 U-Net Restoration이 가장 효과적입니다(YOLOv8m Blur mAP: 0.432 → 0.640). 단, 저조도나 Noise가 혼합되는 환경이라면 Restoration이 역효과를 낼 수 있으므로, Corruption Augmentation이 더 안전한 선택입니다. 이상적으로는 입력 영상의 열화 유형을 분류한 후 조건별 최적 전략을 적용하는 Adaptive Pipeline이 최선입니다. 또한 실시간 요구가 있다면 추론 속도가 빠른 YOLOv8m_aug가 최적 조합입니다.

---

**Q. 프레임워크 내부를 monkey-patching한 것이 좋은 방법인가요?**

> 장단점이 있습니다. 장점은 프레임워크를 fork하지 않고도 원하는 동작을 구현할 수 있다는 점입니다. 기존 학습 파이프라인(mosaic, mixup 등)을 그대로 활용하면서 corruption만 추가할 수 있었습니다. 단점은 프레임워크 버전 업데이트 시 호환성이 깨질 수 있다는 점입니다. 이를 위해 버전을 requirements에 고정하고, 업데이트 시 반드시 테스트하는 프로세스가 필요합니다. 실험 환경에서는 효율적인 방법이었지만, 프로덕션 배포라면 공식 인터페이스가 제공될 때까지 기다리거나 fork 방식을 고려할 것입니다.

---

### 상황형 질문

**Q. 결과가 기대와 다르게 나왔을 때 어떻게 했나요?**

> Image Restoration이 Noise에서 오히려 성능이 크게 하락했을 때입니다. 처음에는 모델 학습 문제로 의심했지만, 복원 품질 자체는 PSNR 34dB로 충분히 높았습니다. 복원된 이미지를 직접 시각적으로 확인해보니 노이즈가 제거되는 동시에 객체의 경계가 뭉개지는 것을 발견했습니다. 이를 over-smoothing으로 정의하고, 소형 객체에서 특히 치명적인 이유를 분석했습니다. 결과적으로 이 "실패"가 전략의 trade-off를 규명하는 핵심 발견이 되었고, 보고서의 가장 중요한 인사이트 중 하나가 되었습니다.

---

**Q. 이 프로젝트에서 혼자 하기 힘들었던 점은?**

> 52회 평가를 수행하는 데 필요한 GPU 시간이 가장 큰 제약이었습니다. RTX 3070 Ti 8GB로 모든 실험을 진행했는데, 특히 Faster R-CNN 학습에 26시간 이상이 소요되었습니다. 이를 해결하기 위해 모델 순차 로딩 후 GPU 메모리를 즉시 해제(`cuda.empty_cache()`)하고, 전체 평가 파이프라인을 자동화하여 대기 시간을 최소화했습니다. 또한 실험 우선순위를 정해 가장 의미 있는 비교부터 수행하는 방식으로 리소스를 효율적으로 배분했습니다.

---

**Q. 이 프로젝트를 더 발전시킨다면?**

> 세 가지 방향을 생각해볼 수 있습니다. 첫째, **Adaptive Pipeline** 구현입니다. 입력 이미지의 열화 유형을 분류하는 경량 classifier를 추가하여, 조건별 최적 전략을 자동으로 선택하는 시스템을 구축할 수 있습니다. 둘째, **열화 다양화**입니다. 안개, 비, 야간, JPEG 압축 등 실전 드론 운용 환경의 추가 열화 조건을 포함하면 실용성이 높아집니다. 셋째, **Augmented + Restoration 조합**입니다. Augmented 모델에 Restoration을 전처리로 추가하면 Noise와 Blur 모두에서 최적 성능을 낼 가능성이 있는데, 이를 실험하지 못한 것이 아쉽습니다.

---

**Q. 한계점이 있다면?**

> 세 가지 주요 한계가 있습니다. 첫째, 열화 파라미터가 단일 severity로 고정(sigma=15, kernel=9)되어 mild/severe 다양한 강도에 대한 분석이 부족합니다. 실환경에서는 열화 강도가 다양하므로 severity-aware 분석이 필요합니다. 둘째, 열화 유형이 3종으로 제한되어 안개, 비, 야간 등 방산 특화 조건이 빠져 있습니다. 셋째, Augmented + Restoration 조합 전략을 실험하지 못했습니다. 이 한계들을 인식하고 있으며, 향후 연구에서 보완할 방향을 명확히 하고 있습니다.
